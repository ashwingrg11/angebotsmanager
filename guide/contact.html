# Contact

A global list of Contact is maintained. These contacts can be assigned to Clients and Partners which form the basis for adding Projects and Offers . Take note that without assigning a Contact to a Client , a Project cannot be created as the Select Client Contact field while adding Project is a mandatory field. Further without assigning a Contact to a Partner, an Offer cannot be created as a field namely Primary Partner Contact , is a mandatory field. The list which populates this field is that of the Contacts assigned to the Partners. Additionally, the contacts are also used for Email Communication. Almost all the emails (except for the Offer Extension Emails) generated by the system are sent to the Primary Partner Contact with the sender address being that of the JI User assigned to the Offer. But the Offer Extension Emails are sent by the Primary Partner Contact to the JI User. This is further discussed in the Email Templates section.

How to add a Contact?

To add a contact, simply click on the Contacts button located on the navigation menu. This results in a dropdown with two options.

  • Add Contact

  • List/Edit Contacts

Click on the Add Contact to open a page where fields used for the contact are displayed. Enter the appropriate data and click Add button.

# Fields

  • Select Language (Required) - A dropdown field to select the language of the Contact. This dropdown is populated as per the languages that exists in the system

  • Select Title (Required) - An dropdown field to select a title for the Contact. This field is dependent on the language chosen. If the language for the contact is chosen as English, the title field is refreshed to hold Mr./Mrs./Ms. If Germany is chosen as the language for the contact, the title field will hold Herr/Frau. For any other languages, the title field will hold Mr./Mrs/Ms.

  • Greeting (Required) - A text field where Greeting for the contact is entered.

  • First Name (Optional) - A text field to enter the first name of the contact.

  • Last Name (Required) - A text field to enter the last name of the contact.

  • Company (Required) - A text field to enter the Company of the contact.

  • Department (Optional) - A text field to enter the Department of the contact.

  • Job Title (Optional) - A text field to enter the Job title of the contact.

  • Email (Required) - A text field to enter the email address of the contact. This field is significant for Email Communication. Take note that the data in this field should be unique. In other words, contact with same email address cannot be added twice.

  • Phone (Optional) - A text field to enter the Phone number of the Contact.

  • Mobile Phone (Optional) - A text field to enter the Mobile number of the Contact.

  • Note (Optional) - A text field to enter Note (if any) for the contact

Leaving the optional fields empty will not generate any validation errors and a user can proceed to add the projects.

# List/Edit Contact

This page will list all the contacts that have been added to the system. The contents of this page are in tabular form which contains columns whose brief description have been mentioned below

# Columns of the List Contact page

The Columns that appear in the List Contact page are

  • First Name - The first name of Contacts appears in these columns. Since first name is an optional field while adding contacts, so if the first name had not been entered, N/A is seen in this column for those contacts whose first name had not been entered.

  • Last Name - Lists the last name Contacts

  • Email - Lists the email addresses of each contacts that have been added.

  • Partners - Lists the Partners that the contacts have been assigned to. May contain the names of more than one Partner as multiple Partners can be assigned same Contact repeatedly.

  • Clients - Lists the Clients that the contacts have been assigned to. May contain the names of more than one Client as multiple Clients can be assigned same Contact repeatedly.

  • Projects - Lists the name of the Projects that the Contact is associated with. May contain the names of more than one Project as multiple Project can be associated with the same Contact repeatedly

  • Action - Contains options which enables a user to edit/delete a Contact

User can see two icons under the Action column whose description have been mentioned below

 The edit button is used to edit the existing Contact details. Same validation rules are applied while editing Contact i.e. the required fields need to be filled with appropriate data.

 The delete button is used to delete a Contact. However, do note that if a Contact is being used in other modules such as Client , the contact cannot be deleted. So, in order to delete a Contact, a user first needs to delete the respective module associated with the Contact and then proceed to delete the corresponding Contact.
