# Offer

Offer(s) is the most used module in the JI Offer tool system. This module is directly dependent on the Projects, Partners, Microsites Reporting Summary, Partner Emails.

How to add an Offer?

To add an Offer , click on the Offers button located on t[he navigation menu. This results in a dropdown with three options.

  • Add Offer

  • List/Edit Offers

  • Offer-Offer Types

Clicking on the Add Offer will load a page where fields used for the Offer are displayed. Enter the appropriate data and click Save Offer button. However, the fields used in this module extend to four pages. The fields for Offer are included within tabs just below the breadcrumb. There are four tabs namely -

  • General Offer Information

  • Detailed Offer Information

  • Admin Information

  • Communication Package

# Fields

Discussing all the fields for this module might prove to be quite tedious for the scope of this manual and also for the user(s) to go through it all. So, we will only be discussing the required fields within this module to ease the user(s) to go through it.

# Required Fields

  • Select Project (Required) - A single select field to select Project whose Offer needs to be created. Selection of the Project determines the Partner, Placement and thus the Partner Contact for the particular Offer.

  • Offer Title (EN) (Required) - Enter the desired name of the offer in English.

  • Offer Title (DE) (Required) - Enter the desired name of the offer in German.

  • Select Placement (Required) - A multiple select field which lists the names of the Placement(s) for the Project so selected. Notice that there is a ' ' sign besides the Select Placement label. Clicking on this sign will display a modal pop up in which a new Placement for the Project can be created if it has not been done so as of yet.

  • Select Offer Partner (Required) - A dropdown field to select the Offer Partner of the Project so chosen. This field is populated based on the Partner associated with the particular Project.

  • Primary Partner Contact (Required) - A dropdown field to select the Partner Contact of the Offer Partner so chosen. This field is populated based on the Partner Contact assigned with the particular Partner. This too has a ' ' sign besides the Primary Partner Contact label. Clicking on this sign will display a modal pop up in which a Partner Contact can be assigned to any Partner if he/she has not been assigned so as of yet.

  • Valid From (Required) - A datepicker field to choose the start date of an Offer. Clicking on this field will display a calendar from which a user can choose a date from which the Offer is to start.

  • Valid Until (Required) - A datepicker field to choose the end date of an Offer. Clicking on this field will display a calendar from which a user can choose a date at which the Offer is to end.

  • Select JI Contact (Required) - A dropdown field to choose the JI contact of the particular Offer. This field is populated based on the Project so selected at the beginning while creating an Offer. The JI Contact are basically the user(s) of Offer Tool. In essence, the email addresses of the JI contact so selected for the particular Offer are used as the Sender's email address (for all emails except extension email) in email communication and to that of recipient's address in case of Offer Extension email only.

  • Communcation Package (Pre-defined) - This page has the option to set the communication package to 'Yes' or 'No'. Setting the Communication Package to "Yes" will generate emails for Activation Links, Microsites and also the Microsite Summary Details. However, setting it to "No" will not generate any of the above mentioned emails or reporting details. Please note that by default, Communication Package is set to No while creating an Offer.

# List/Edit Offers

This page will list all the Offers that have been added to the system. The contents of this page are in tabular form which contains columns whose brief description have been mentioned below

# Columns of the List Offer page

The Columns that appear in the List Offer page are

  • Offers - The name of the Offer(s) appears in these columns.

  • Projects - Lists the Project associated with the Offer. An Offer can only be associated with a single Project.

  • Placements - Lists the Placement(s) associated with the Offer. An Offer can have multiple Placements

  • Market - Lists the Market detail so entered when creating an Offer

  • Valid From - Lists the start date of an Offer.

  • Valid Until - Lists the end date of an Offer.

  • Communication Package - Displays the option of the Communication Package of the Offer

  • Action - Contains four buttons whose functionalities have been discussed below.

A user will see four buttons against each Offer name at the end of each row in this page.


  The edit button is used to edit the existing Offer details. Same validation rules are applied while editing Offers. Below mentioned are some instances, a user needs to be aware of while changing the Communication Package when editing the Offer
- Lets assume that initially an offer is created and the Communication Package is set to 'Yes'. Corresponding Microsite(s) are generated for the specific offer. Lets also assume that the user(s) have filled in the data of the Microsite(s). - Now when the communication package of the particular Offer is edited and set to 'No', the Microsite(s) of that Offer are not removed from the Microsite Feedback Summary page but their links no longer work. Their data(s) are only viewable on the table of Microsite Feedback Summary page. Consequently, setting the Communication Package of that Offer back to 'Yes' will result in the loss of data(s) of the Microsites and new Microsites are created.


  The delete button is used to delete an Offer. However, do note that if a Offer is being used in other module, the Offer cannot be deleted. So, in order to delete an Offer that is been used in other modules, a user first needs to delete the respective modules and then proceed to delete the corresponding Offer.


  Clicking on this icon will enable the user to Edit Placement of the particular Offer associated with a specific Project. Clicking on this button will take the user to a page where the particular Placement can be edited.


  Clicking on this button will disable an Offer and that particular Offer will no longer be visible on the List Offer page. Also there is no way to enable that Offer. So, a user needs to be aware of this fact and proceed to disable the Offer.

# Access Rights

Access Rights for a creating/editing and deleting a Partner depends on the user and user type.This is discussed below.

  * Admin User

   An admin user can view/create/edit and delete any Offers irrespective of the fact that he/she is assigned to the Offer or not

  * General User

   A General User can only view/create/edit Offer for the Projects that he/she is assigned to. He/she cannot view the Offers of the Projects he has not been assigned to. A general user does not hold the right to delete an Offer.

  * Viewer

   A Viewer can see all the Offer(s) name but only certain description of the Offer(s) so clicked
