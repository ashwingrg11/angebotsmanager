# Clients

Client is the company that places an order for a project. It is very simple, with just formal information about their name, address and contacts within that company etc.

# How to add a client?

The clients can be added by going into the Additional Information and then clicking on the Clients which will collapse the menu. There will be two options

  • Add Clients
  • List/Edit Clients

Clicking on the Add Client will redirect you to a page, where a user can see fields. The fields that are required are mentioned below. When appropriate data have been entered, click on the Add button to confirm the entry of client.

# Fields

The fields seen when adding a client are

  • Client Name (Required) - Enter an appropriate name for the Client

  • Region (Optional) - Enter the appropriate region for a client

  • Notes (Optional) - Enter appropriate note (if any) for a Client

* Click to enlarge

Please note that of these three fields, only the Client name is a required field. Rest of the fields are optional.

# List Clients

This page will list all the clients that have been added to the system. A user will see three icons against each client name at the end of each row in this page.

    The edit button is used to edit the existing Client details. Same validation rules are applied while editing clients i.e. only the Client Name is a required field.

    The delete button is used to delete a client. However, do note that if a client is being used in other module, namely Project , the client cannot be deleted. So, in order to delete a client that is been used in Project, a user first needs to delete the respective Project and then proceed to delete the corresponding client.

    Clicking on this icon will enable the user to assign Contacts to the corresponding client. After clicking this button, in the corresponding page that loads, a user needs to tick mark the appropriate contact which signifies that the said contact has been assigned to the Client. Important factor to note here is that assigning contacts to the client is compulsory as not doing so will generate a validation error while adding Project. Without assigning the contact to a client, a validation error will be generated while adding Project asking a user to select the Client Contact. So before adding a Project, a user needs to assign contact to the corresponding client.

# Client's Detail summary page

Client details summary page shows the detailed information about the client. In other words, this page shows the data that was entered for the particular client. This page can be accessed by clicking on the client name in the List/Edit Client page.

# Access Rights

Please note that the clients cannot be added by a General User and a Viewer. Also these users cannot view the client's list. A General User can only see the the client of the Project that he/she is assigned to
