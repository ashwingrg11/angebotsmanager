# Users

User(s) of the Angebotsmanager are generally the system accessors of the tool. These are the entities who can log into the system and make use of the various features provided. They can be assigned to Projects and Offers. They are also the entities whose email addresses are used in Email communication. Basically, there are three types of User(s) in Offer tool who have been briefly described below -

  • Admin User - Admin user(s) are the major user(s) of Offer tool. Basically, they have the right to view all the data(s) inserted, edit the existing data(s), assign contacts and also delete any entity (provided they have no dependencies). They are able to set roles for other users.

  • General User - A general user is the secondary user of the offer tool. A secondary user does not mean that the General User has lesser priority as compared to that of an Admin User, but its just that he/she will not have access to some features which are available to an Admin user. For instance, he/she is not able to delete anything. They can only view the Project(s) and Offer(s) which are assigned to them, unlike the Admin Users who can view all the details. They can add the following entities - Offers, Placements, Partners and Contacts. However, they do not have the right to add a Project, Channel, Offer-Offer types and Email templates. Though they can edit and view these modules (provided they have been assigned to these modules). A general user does not have the right to add other user(s)

  • Viewer - A viewer is a user who can view only certain portions of all Offers. This type of User is basically just a spectator with almost no functionality except that of viewing data of an Offer. He/She is not able to add/edit/delete anything

# How to add a User into the system?

User(s) can be added only by an Admin user. To add a User into the system, click on Additional Information -> Users -> Add User on the left hand navigation menu. Enter the appropriate details in the fields and click Add button. Below described are the fields seen in the Add User page.

# Fields

The fields seen when adding User(s) are

  • First Name (Required) - First name of the user is to be inserted in this field

  • Last Name (Required) - Last name of the user is to be inserted in this field

  • Email (Required) - Provide appropriate email address for the user in this field. The provided email address will be used to log into the system and also in email communication

  • Telephone (Optional) - A field to insert the telephone number of the User

  • User Type - A single select dropdown field to select the User type. User type as mentioned above are of three types - Admin, General and a Viewer

  • Username (Required) - A field to enter the Username for the particular user

  • Password - A field for inserting the password. This is particularly significant as the email address and password of the users are required to log into the system

  • Confirm Password - A field for inserting the password again for measures of extra security and to ensure that the password inserted by the user is deliberate and has not mispelled it. If the two password fields do not match, this page cannot be submitted.

List Users

This page lists all the users that have been added to the system. This page is in tabular form whose columns have been briefly discussed below -

# Columns of the List User page

The Columns that appear in the List Users page are

  • First Name - Displays the first name of the User(s)

  • Last Name - Displays the last name of the User(s)

  • Email - Displays the email addresses of the User(s). Note that clicking on these email addresses will show the User Details Summary Page

  • User Type - Displays the User Type of the User(s)

  • Project - Displays the names of the Project(s) that the user is assigned to. A single user maybe associated with multiple Projects

Users can see two icons under the Action column whose description have been mentioned below


 The edit button is used to edit the existing User details. Same validation rules are applied while editing User as that of adding User except for the two fields of Password. When editing an existing User, the password field is not populated allowing the admin users to reset the password. Note that only an admin user can edit the user details.

 The delete button is used to delete a User. However, do note that if a User is being used in other modules such as Project, then he/she cannot be deleted. Only an admin user possesses the right to delete a user.

# Access Rights

Please note that a User can be created by an Admin User only. a General User and a Viewer do not have access to these features. Also these users cannot view the User(s) list directly. A General User has the ability to Select the existing User list when adding Project and Offers.
