# Projects

Projects are limited in number and commissioned by Client Contacts. Project will be separate for a particular client. Access rights for the projects are also limited depending on the type of user which will be discussed at the end.

# How to add a Project?

The Projects can be added by going into the Additional Information and then clicking on the Projects which will collapse the menu. There will be three options

  • Add Project

  • List Projects

  • Project Categories

(Take note that clicking on Additional Information navigation button will display the List Project page by default.)

Clicking on the Add Project will redirect you to a page, where a user can see fields. The fields that are required are mentioned below. When appropriate data have been entered, click on the Add button to confirm the entry of project.

# Fields

  • Project Name (Required) - A field to enter the name of the Project

  • Select Project Logo (Optional) - An option to select a logo for the Project

  • Select Client (Required) - The dropdown is populated as per the clients entered

  • Project Description (Optional) - Description for the Project

  • Select Client Contact (Required) - This field is automatically refreshed when a client is selected. The list shows the name of the Contact which is assigned to a project

  • Select JI Contact (Required) - Assign a user to a Project. This field determines which users have access to the particular project based on their roles

  • Select Categories (Optional) - A field to Select Categories for a Project

  • Language (Required) - Select the language for the Project. Please note that this field is used when creating Offer which will be discussed in the same section

  • First Issue Launch Date (Optional) - Optional field which states when the offer was created

  • Circulation (Optional) - Optional field to enter the Circulation details of the Project.

  • Note (Optional) - Optional field to enter any note associated with the Project

* Click to enlarge

Leaving the optional fields empty will not generate any validation errors and a user can proceed to add the projects.

# List Project

Clicking on the Additional Info. will display a list view of all the Projects that exists in the system, by default. Clicking the same will also collapse a menu with various options. Further, the list view of the Project can also be displayed by clicking on the Projects > List Projects. This page will list all the projects that have been added to the system. The contents of this page are in tabular form which contains columns whose brief description have been mentioned below

* Click to enlarge

# Columns of the List Project page

The Columns that appear in the List Project page are

  • Project Name - The names of each Project appears in these columns

  • JI Contacts - Contains the name of JI Contacts that have been selected while adding a Project

  • Placements - Contains the name of the Placements that have used the particular Project

  • Action - Contains options which enables a user to edit/delete a Project

Users can see two icons under the Action column whose description have been mentioned below

   The edit button is used to edit the existing Project details. Same validation rules are applied while editing Projects i.e. the required fields need to be filled with appropriate data.

   The delete button is used to delete a Project. However, do note that if a Project is being used in other modules such as Channel , Offer , to name a few, the project cannot be deleted. So, in order to delete a project, a user first needs to delete the respective module associated with the project and then proceed to delete the corresponding Project.

# Project Categories

This module defines the Project Categories for a Project and finds it's use while adding a Project. Project categories can be added by clicking on Project > Project Categories. On the page that loads, enter the appropriate category and click on Create Category to create a Project Category. Below this field, a list view of all the categories are seen in tabular form which can be edited/ deleted. The Project categories can be deleted irrespective of the fact that they are being used in other modules. The images below illustrates the use of Project Category within a Project.

# Project Detail View

Clicking on the Project name in the List Project page will give a detailed view of the particular project. It will provide a summary of the data(s) entered in the particular project. On this page, a user will also see a Project Logo (if it has been added) and an option to delete the logo. At the top of the Logo image, there exists a hyperlink Delete Logo clicking on which will prompt a user to confirm the deletion. Upon confirming, the logo is removed for the Project

# Access Rights

Access Rights for a particular user are defined based on the JI Contacts selected in a Project. In other words, the user and user type determine if the Project details and modifications are available to the user or not.

  * Admin User

   An admin user can see all the Projects irrespective of the fact that he/she is assigned to the Project or not

  * General User

   A General User can only see the Projects that he/she is assigned to. He also has access to view the Detail Summary page of the Project he is assigned to or any relevant data associated with the Project.

  * Viewer

   A Viewer cannot see the Project details or modify them. He/she can only see the projects which are included in Offers
