# Email Templates

Basically, Email Templates are the templates created by the user for any outgoing mail. Angebots Manager allows a user to create email templates for different types of emails generated by the system. Moreover, the language of the email templates can also be pre-defined. There are, in essence, six types of emails and a user can create a template for each of these type. Email Template is nothing but the skeleton structure for the emails that are to be sent. The email so sent strictly adheres to the template. Important Points to note here is that

  • Email Templates are generated based on the Project. The Project and email type should be unique which means that a user cannot create two templates with the same Project and same email type.

  • If an email template is not defined before creating an Offer, the outgoing emails will have default values stored in its body. To add a template structure for these outgoing emails, a user needs to individually edit the emails in List/Edit Queue Emails section and add the desired data(s) into those emails. Editing all the emails of the same Project at the same time is not possible.

  • The language of the email templates are pre-defined and the template(s) are generated based on the language so selected

So, before creating an Offer, a user needs to create an email template if he intends the email to contain some basic data(s) before it is generated. The different email types are listed below

  • Offer Activation - The email generated when an Offer is created.

  • Offer Extension - These are emails which are automatically sent (when Primary Partner Contact chooses to extend the Offer from Microsite 1 or Microsite2), with the JI User as the recipient of the Offer Extension email. The extension itself is then a manual process.

  • Microsite1 - The email generated at the start of every month ( by Default ) till the Offer validity period.

  • Microsite2 Final - The Microsite2 Final email are the final email generated at the end of the Offer validity period. These emails are generated at the start of the following month of the last month (by Default) of Offer validity period. For instance, if the Offer Validity period is upto September, then these emails are generated on 1st of October

  • Microsite2 Reminder - This email is generated automatically by the system but not sent. By default, it is not ticked mark i.e. sent out. Users have the option to decide whether to send out this email or not. It will be further discussed in List/Edit Queue Emails section

How to add an Email Template?

To add a template for an email, click on the Partner Emails > Email Templates. There a user can see two options listed as below.

  • Add Template

  • List/Edit Template

Click on Add Templates to add a template for an email based on their types.

# Fields seen within the Add Template

The fields seen in the Add Templates page have been briefly described below

  • Select Project (Required) - A dropdown to Select Project for which the email template is to be generated. The existing Projects of the system populates this dropdown list

  • Select Type (Required) - A dropdown to select the email type that is generated by the system

  • Sender Email (Required) - A text field to enter the sender's email address. By default, if a template is not created, then the JI User's email address is used for sender's address

  • Subject (Required) - A text field to enter the Subject of the Email

  • CC Emails (Optional) - A text field to enter the CC email of the recipient. Clicking on  CC will produce another field to enter CC Emails for the template. Similarly, multiple CC emails can be inserted for the template

  • Attach a File (Optional) - A link to attach the desired files for the emails to be sent. When a file is attached, a user can see icon just next to it, which is used to remove the attached file.

  • Variables (Optional) - Variables seen enclosed within {}. These variables can be copied and pasted in the body section and as the name enclosed within the {} suggests, the emails are sent containing the different variables. Please note that the variables need to be copied in the same format as given above i.e. within {}.

  • Template Content (Required) - The body for the email to be sent. User can type in the text required or also copy the variables from above

# List/Edit Templates

This page lists all the templates that is created for the system.

# Columns of the List Email Templates page

The major columns that appear in the List Offer-Type page are

  • Project Name - The names of each Project for which email templates are defined

  • Template Type - Lists the types of emails for which templates are created

  • Sender Email - Lists the email addresses of the senders

  • CC Emails - Lists the CC emails if entered

  • Subject - List the subjects for each template if it has been entered

  • Action - List the various actions available to the user. Each icon has been elaborated below

  The eye icon is used to view the details of the Template created. It gives a brief overview of the data(s) entered into the email templates

  The edit button is used to edit the Email template. Same validation rules are applied while editing Email Template as that used while Adding Templates.

  The delete button is used to delete an Email template. Any Email Template can be deleted irrespective of the fact that they are being used in other modules.

# Access Rights

Please note that an Email Template can be viewed/created/edited and deleted by an Admin User only. General Users and Viewers . do not have access to these features.
