# Communication Package

Communication Package defines the number of days prior to or after which the emails generated should be sent by the system. It is Project Specific which means that the emails that are generated by the system is sent based on a particular project or an Offer so created. If communication package is not defined, the emails are sent on the default dates as mentioned below.

# Default Date at which Emails are sent.

  • Activation Link - By default activation link are sent 5days prior to the first Placement Start date within the Offer.

  • Microsite1 - By default, Microsite1 emails are sent at the first day of each month till the Offer Validity Period.

    • Microsite2 - By default, Microsite2 emails are sent on the first day of the following month after the final month of Offer validity period. It might seem tricky but here is an example to further illustrate the matter. For instance, an Offer ends on February 2016. Then, Microsite2 email is sent on the first day of the following month at which the Offer expires i.e. it is sent on 1st March 2016

  • Microsite2 Reminder - By default, Microsite Reminder emails are sent 2 days after the Microsite 2 final email has been sent.

So, in conclusion, the days for each of the above defined email can be changed by defining the communication package. Please note that Communication Package needs to be defined before creating an Offer. Defining the same after an Offer has been created will yield no results and the emails will be sent by default.

# How to add Communication Package?

To define Communication Package, click on Partner Emails > Communication Package > Add Communication Package which will display fields. Each of these fields have been briefly described below

* Click to enlarge

# Fields

  • Select Project (Required) - Select the desired Project for which communication package is to be defined

  • Prior Duration(days) Activation Email - Define the number of days prior to first placement start date for which Activation email is to be sent

  • After Duration(days) Microsite1 Monthly Email - Define the number of days after which Microsite 1 email is to be sent

  • After Duration(days) Microsite Final Email - Define the number of days after which Microsite 2 final email is to be sent

  • After Duration(days) Microsite Reminder Email - Define the number of days after which Microsite 2 reminder email is to be sent

# List/Edit Communication Package

This page lists all the Communication Packages that were defined for email types.

# Columns of the List Communication Package

The major columns that appear in the List Communication Package page are

  • Projects - The names of each Project for which the emails were created

  • Offer Activation (Days) - Lists the number of days Activation Link was intended to be sent prior to the first placement start date within an Offer neglecting the default send out dates.

  • Microsite1 Monthly (Days) - Lists the number of days after which Microsite1 email is intended to be sent neglecting the default send out dates.

  • Microsite2 Final (Days) - Lists the number of days after which Microsite2 Final email is intended to be sent neglecting the default send out dates.

  • Microsite2 Reminder Email (Days) - Lists the number of days after which Microsite2 Reminder email is intended to be sent neglecting the default send out dates.

  • Action - Lists the various actions available to the user. Each icon has been elaborated below

 

  The edit button is used to edit the Communication Package details. Same validation rules are applied as that used while Adding Communication Package.


  The delete button is used to delete any Communication Package.

# Access Rights

Please note that a Communication Package can be viewed/created/edited and deleted by an Admin User only. General Users and Viewers do not have access to these features.
